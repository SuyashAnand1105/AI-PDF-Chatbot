🤔 WHAT IS THIS PROJECT?

This is an AI-powered chatbot that lets you upload PDF documents and
ask questions about them. Think of it like ChatGPT, but specifically
for your PDF files!

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🎯 WHAT CAN IT DO?

1. Upload PDFs
   → You can upload any PDF document (textbooks, research papers,
     reports, manuals, etc.)

2. Chat with AI
   → Ask questions about the PDF in plain English
   → AI reads the PDF and answers your questions
   → Only answers based on what's IN the PDF

3. Smart AI Tools
   → Summary: Get a quick overview of the entire document
   → Keywords: Extract important terms and concepts
   → Important Points: Key takeaways from the document
   → Quiz: Generate multiple-choice questions to test yourself
   → Flashcards: Create study cards for memorization

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🏗️ HOW DOES IT WORK? (Simple Explanation)

Step 1: You upload a PDF
   → The file is saved on your computer

Step 2: Backend processes the PDF
   → Extracts all the text from the PDF
   → Breaks the text into small chunks (like paragraphs)
   → Stores these chunks in memory

Step 3: You ask a question
   → Your question is sent to the backend
   → Backend searches for relevant chunks that match your question
   → Sends these chunks + your question to the AI

Step 4: AI generates an answer
   → AI reads the relevant chunks
   → Understands your question
   → Writes a helpful answer
   → Sends it back to you

Step 5: You see the answer
   → Answer appears in the chat interface
   → You can ask follow-up questions
   → AI remembers the conversation context

🔧 TECHNICAL EXPLANATION (For Developers)

FRONTEND (What you see):
   • Built with React.js (modern JavaScript framework)
   • Styled with Tailwind CSS (utility-first CSS)
   • Uses Vite (fast build tool)
   • Zustand for state management (simpler than Redux)
   • Framer Motion for smooth animations
   • React Router for navigation between pages

BACKEND (Behind the scenes):
   • Built with Node.js + Express.js
   • Single server.js file (all-in-one approach)
   • In-memory storage using JavaScript Maps
   • No database needed (MongoDB removed for simplicity)
   • JWT authentication (secure login tokens)
   • Bcrypt for password encryption
   • Multer for file uploads
   • pdf-parse to extract text from PDFs

AI INTEGRATION:
   • Uses OpenRouter API (free tier)
   • NVIDIA Nemotron 70B model (powerful and free)
   • Simple keyword-based search (no vector embeddings)
   • Chunks text into ~1500 character pieces
   • Finds relevant chunks using keyword matching
   • Sends context + question to AI
   • AI generates answer based only on provided context

AUTHENTICATION:
   • User registers with name, email, password
   • Password is hashed with bcrypt (secure)
   • JWT token generated on login (expires in 7 days)
   • Token sent with every request to verify identity
   • Users stored in-memory Map (id → user object)

PDF PROCESSING:
   • PDF uploaded via multipart/form-data
   • Saved to backend/uploads/ folder
   • pdf-parse extracts raw text
   • Text cleaned and split into chunks
   • AI generates summary, keywords, important points
   • Status changes from "processing" → "completed"

CHAT SYSTEM:
   • Each chat linked to one or more PDFs
   • Messages stored in-memory (user + AI messages)
   • Last 6 messages sent as conversation history
   • Relevant chunks found using keyword search
   • AI prompt includes: history + context + question
   • Response streamed back to frontend

📊 DATA FLOW:

User Action → Frontend → API Request → Backend → Process → AI API
                ↑                                              ↓
                └──────────── Response ←──────────────────────┘

Example: Asking a question

1. User types: "What is this document about?"
2. Frontend sends POST to /api/chat/:id/message
3. Backend receives request
4. Backend finds relevant PDF chunks
5. Backend builds AI prompt with context
6. Backend calls OpenRouter API
7. AI generates answer
8. Backend sends answer back
9. Frontend displays answer in chat
10. User sees the response!

🔄 API ENDPOINTS:

AUTH:
   POST /api/auth/register    → Create new account
   POST /api/auth/login       → Login with email/password
   GET  /api/auth/me          → Get current user info

PDF:
   POST   /api/pdf/upload     → Upload a PDF file
   GET    /api/pdf            → Get all user's PDFs
   GET    /api/pdf/:id/status → Check processing status
   DELETE /api/pdf/:id        → Delete a PDF

CHAT:
   POST   /api/chat/create    → Create new chat session
   GET    /api/chat           → Get all user's chats
   GET    /api/chat/:id       → Get specific chat
   POST   /api/chat/:id/message → Send a message
   DELETE /api/chat/:id       → Delete a chat

AI TOOLS:
   GET /api/ai/summary/:pdfId         → Generate summary
   GET /api/ai/keywords/:pdfId        → Extract keywords
   GET /api/ai/important-points/:pdfId → Get key points
   GET /api/ai/quiz/:pdfId            → Generate quiz
   GET /api/ai/flashcards/:pdfId      → Create flashcards
